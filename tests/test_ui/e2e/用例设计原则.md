（二）端到端业务流用例（test\_e2e）

1. 设计原则

   跨模块串联：一条用例串多个模块，验证业务闭环

   核心主流程优先：只做高价值链路，不做分支流程

   复用 biz 层能力：不重复写单模块逻辑

   自带完整环境初始化与清理
2. 组织方式

   按业务域 / 业务线建文件，而非按模块：

   test\_system\_permission\_e2e.py（系统权限全链路）

   test\_user\_role\_menu\_e2e.py（用户 - 角色 - 菜单授权链路）

   用例命名：test\_e2e\_业务线\_场景

   例：test\_e2e\_permission\_assign\_and\_verify
3. RuoYi 核心 E2E 链路示例（权限分配全流程）

   业务流：登录 → 创建角色 → 分配菜单 → 创建用户并绑定角色 → 校验用户可见菜单
